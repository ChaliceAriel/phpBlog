Meet my New Friend Laravel:
The Modern, Feature-rich Framework for PHP

When my boyfriend recently asked me what I planned to do this weekend, I told him, "Play with Laravel." My boyfriend is a very experienced PHP developer, but he jokingly asked, "Who is Laravel?" I smiled and responded playfully, "My beautiful new friend"!

What makes Laravel beautiful? Laravel simplifies the process of building complex applications. It has become one of the most popular modern PHP frameworks, if not the most popular. It has built-in support for user authentication and authorization, making it easy to write secure applications that are scalable. Laravel bundles allow you to quickly add features to your application, much like npm packages for NodeJS. Laravel's 'Query builder' allows you to programmatically create SQL queries using built-in classes and methods. Tools like Laravel Forge make application deployment simple, allowing you to focus on writing code. Finally, Laravel is well-maintained, with a large, friendly, and dedicated community supporting it. Laravel even has its own annual developer conference, Laracon, and a fantastic training site: laracasts.com.

Setting up Your Laravel Development Environment in the Cloud

One way to quickly get up and running with Laravel is to set up a cloud-based development environment using a service like Cloud 9, Codeanywhere, or Koding. Codeanywhere offers Laravel preconfigured development right out of the box. Here is a post that walks you through installing Laravel on Koding.com through your terminal, and I will go over setting up Laravel with Cloud 9 in the steps below.

  • 1. Sign-up for the Free-tier development environment using your email address. After selecting a username and answering some questions, Cloud 9 will ask for your credit card information. They will not charge you as long as you remain on their free plan.
  • 2. Follow the link in the email sent by Cloud 9, and set the password for your account.
  • 3. Click on 'Create New Workspace' in the center of the gray box on your Cloud 9 Dashboard. Type in the name of your project in the 'Workspace name' field, and add some details about your project in the 'Description' field. Here is a video on creating a workspace.
  • 4. Make sure you select the PHP template at the bottom of this page, then click on the green 'Create workspace' button.

  • 5. Now it's time to install Laravel! At the bottom of your new workspace is a tab with a bash terminal. If you don't see it, open a new bash shell by clicking on 'window' in the menu, and selecting 'new terminal' from the dropdown.
  • 6. Follow this installation guide step-by step, copying the code and pasting it into the terminal. I've also included these instructions below:
    • rm README.md php.ini hello-world.php
    sudo composer self-update
    composer createproject laravel/laravel ./laravel --prefer-dist
    shopt -s dotglob
    mv laravel/* ./
    rm -rf laravel
  • 7. Copy and paste this code into the workspace terminal. It takes a couple minutes for Laravel to finish downloading. When it's complete, you should see these three lines at the bottom of your terminal: If you don't see these three lines, Laravel was not successfully installed. Copy the last few lines from the code block above (starting with "shopt..") and paste them into the terminal again.
  • 8. Once you see those three lines, hit Enter on your keyboard and your project file structure will appear in the left column of your workspace.
  • 9. Cloud 9 serves it's files from the root workspace directory, but Laravel serves it's files from the "public" directory, so we need to change a couple more things.
  • 10. Access the apache web server configuration file with the nano text editor by pasting this code into your terminal: sudo nano /etc/apache2/sites-enabled/001-cloud9.conf and pressing 'Enter'. Then edit this file by setting the default directory to the public folder.
    Change this line at the top:
    DocumentRoot /home/ubuntu/workspace
    To this:
    DocumentRoot /home/ubuntu/workspace/public
    To save the file and exit the nano text editor press F2, then 'Y' and 'Enter'.
  • 11. Now you can get the latest version of Laravel by running this command: sudo composer update
  • 12. That's it! Laravel should be up and running now! Click the green 'Run Project' button at the top of your workspace. In the terminal you should see "Starting Apache httpd, serving", followed by a blue web link. Click to open this link and you should see the 'Laravel 5' white splash page.

To set up your database in Cloud 9 and get it working with Laravel, you can follow the last few step in this guide.

Installing Laravel Locally with Your Web Server Stack

This post is not intended as a comprehensive local install guide for Laravel. Installation instuctions will vary depending upon your server stack (i.e. Xampp) and your OS (operating system). Just like in the cloud-based environment above, it is important to set the root folder to the 'public' directory. I started by installing Composer, a popular PHP package and dependency manger required by Laravel's installer, through the Windows command terminal. Because I am already using Xampp for Windows on my desktop, I then followed this excellent tutorial from codementor.io, and it worked like a charm! Well, actually there was one minor hiccup which this tutorial makes note of when it warns:

"...we have to configure our hosts file that allows us to redirect laravel.dev to the localhost that is located in C:\Windows\System32\drivers\etc
IMPORTANT!: To edit this file, maybe we should give access, click properties and allow your user to modify this file."

Basically, if you try to change the hosts file (to add a localhost for our new Laravel virtual host), Windows says that you "don't have permission" to save to that location. There are several solutions to this problem:

  • 1. Right click and run notepad as administrator, then open the file. More on that here.
  • 2. Right click on the file, go to the very bottom of the drop-down and select "properties". Click on the "security" tab and then click the "Edit..." button. In the new modal that pops up, select/highlight your user in the upper window, and then select the checkbox next to "modify" in the lower window. Click "ok" button to save and close the dialog box.
  • 3. Copy(Ctrl+C) the hosts file to the Desktop, edit and save it there, and then paste (Ctrl+V) it back into it's home folder, overriding the original file. This video tutorial shows you how. (Nice try with the security Windows! You've succeeded in making this process a pain, while leaving it easy enough for someone without administrator privileges to change the hosts file.)

After completing the codementor.io tutorial, I now have a virtual host in Xampp for building my Laravel project. But, there is more way I could have installed Laravel...

Laravel Homestead

What is Laravel Homestead? Well, it's a cross-platform PHP development environment that uses Vagrant with Virtual Box. I'll let the folks over at laravel.com explain it:

"Laravel Homestead is an official, pre-packaged Vagrant box that provides you a wonderful development environment without requiring you to install PHP, HHVM, a web server, and any other server software on your local machine. No more worrying about messing up your operating system! Vagrant boxes are completely disposable. If something goes wrong, you can destroy and re-create the box in minutes!"
Wow, they sure sound exited about this! Now I'm excited too!!! Here is video tutorial for setting up Laravel Homestead on Windows 10, and here is an excellent series of videos for mac called the Laravel Homestead Guide.

I can't wait to start building my first app in Laravel, so I'm going to hold off on setting up Homestead just yet. I have Laravel installed and running on Xampp locally, and on my Cloud 9 workspace. If anyone I know is using Homestead, I'd love to hear more about it!
